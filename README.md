# Джобазавр

**Джобазавр** — мультиплатформенное приложение для поиска работы, реализованное как VK Mini App, мобильное приложение (Android/iOS), Telegram-бот и веб-кабинет работодателя. Вся логика и сервисы объединены общей серверной частью.

## Архитектура монорепозитория

- **packages/frontend** — клиент для VK Mini Apps (и web-версии), написан на React + VKUI, с использованием Recoil для стейта и кастомного роутера для VK.
- **packages/app** — мобильное приложение на базе React + Capacitor, поддерживает Android и iOS, использует тот же UI-стек, что и VK Mini App.
- **packages/backend** — основной сервер на NestJS, реализует REST API, авторизацию, работу с БД (MySQL через TypeORM), интеграцию с VK, Telegram, рассылки, админку, задачи по расписанию и др.
- **packages/client-lk-backend** — отдельный сервер для личного кабинета работодателя (NestJS + TypeORM), отвечает за управление вакансиями и откликами.
- **packages/tg-bot** — Telegram-бот на node-telegram-bot-api, интегрирован с общей БД, поддерживает сценарии поиска работы и взаимодействие с web-версией.

## Основные технологии

- **Frontend (VK/Web/Mobile):**

  - React 17, VKUI, Recoil, Vite, Capacitor (для mobile), интеграция с VK Bridge и Telegram WebApp API.
  - Мультиязычность через Tolgee.
  - Интеграция с аналитикой (AppMetrica, Google Tag Manager).
  - Единая кодовая база для VK Mini App и мобильного приложения.

- **Backend:**

  - NestJS, TypeORM, MySQL, Redis, Swagger для автодокументации API.
  - Поддержка авторизации через VK, Telegram, JWT.
  - Модули: вакансии, города, гражданства, профили, события, подписки, уведомления, клубы, юридическая помощь и др.
  - Cron-задачи, рассылки, интеграция с внешними сервисами.

- **Telegram-бот:**

  - node-telegram-bot-api, TypeORM, MySQL.
  - Сценарии: регистрация, сбор контактов, выдача вакансий, интеграция с web-версией.

- **Личный кабинет работодателя:**
  - Отдельный NestJS-сервер, управление вакансиями, откликами, профилями компаний.

## Как развернуть проект

Каждый пакет содержит свои инструкции по запуску (см. package.json и README внутри пакетов). В целом:

1. Установите зависимости во всех пакетах:
   ```bash
   cd packages/<package>
   yarn
   ```
2. Для серверных частей создайте `.env` по примеру `env.example` и пропишите доступы к БД.
3. Запустите нужные сервисы:
   - Frontend (VK/Web): `yarn dev`
   - Mobile: `yarn dev` (или сборка через Capacitor)
   - Backend: `yarn dev`
   - Telegram-бот: `yarn dev`
   - ЛК работодателя: `yarn dev`

## Особенности

- Единая база пользователей и вакансий для всех платформ.
- Гибкая архитектура для масштабирования и интеграции новых платформ.
- Поддержка мультиязычности (RU, KY, TG, UZ).
- Интеграция с VK, Telegram, email-рассылками.
- Расширяемая система уведомлений и подписок.
